## 特性

### CPU 特性

基于8051 指令的8 位单片机；Flash ROM:8K\~48K 字节；RAM:内部256 字节,外部1024 字节；类EEPROM:1024 字节。
宽工作电压范围：Fsys = 16MHZ,VDD = 2.7V\~5.5V。
振荡器：内部高频振荡器48MHZ(精度±2%)，内部低频振荡器800KHZ(精度±2%)。
CPU 主频最大可以达到16MHZ；内置看门狗定时器(WDT)，可软件使能；内建低电压检测功能(LVDC)；支持休眠、空闲两种省电运行模式；可编程代码保护防止代码读出；支持串口在线调试。
低功耗特性：休眠模式5.0V时典型电流200uA；16MHZ、5.0V工作电压下典型工作电流5.0mA。

### 外设特性

多达40 个双向可单独控制的I/O 引脚：高灌/拉电流可直接驱动LED；最多6路电平变化中断引脚；可单独编程配置内部弱上拉、弱下拉、开漏；支持超低功耗唤醒。
A/D转换器：10-bit规格，最大8个通道；内建12/16/24/31路触控按键功能。
多路定时器，可用于计时、输入捕获、方波、单脉冲输出；最多2路独立PWM+4路定时器复用PWM输出。
UART：独立波特率发生器，支持自动波特率校准；4线SPI四种工作模式；硬件I2C主从接口。
LED硬件驱动：支持4×13/5×12/6×11/7×10/9×8/10×7规格共阴/共阳数码管软件扫描驱动。
RTC实时时钟：需要外接32.768KHZ晶振，自带定时与闹钟功能。

## 1.概述

本用户手册涵盖了CBM73xx全系列芯片。芯片内核为8051架构，片上集成FLASH、XRAM、各类外设控制器，外设包含IO、SARADC、SensorADC、PWM、TIMER、LED、RTC、UART、SPI、I2C、WDT、在线调试模块；系统内置POR上电复位、多路内部振荡源。除去VCC、GND、XIN、XOUT外的绝大多数引脚支持多功能复用。

## 2.存储器的构成

### 2.1.程序存储器的构成

CBM73xx采用16位程序计数器，最大寻址48K×8bit程序空间(0000H\~BFFFH)，不可超出寻址边界。复位向量固定在0000H，默认6个中断向量地址：0003H、000BH、0013H、001BH、0023H、002BH；可通过VTOR寄存器配置中断向量整体偏移。

### 2.2.数据存储器的构成

数据存储器分为：805内核256字节片内RAM、1024字节片上XRAM扩展区、SFR特殊功能寄存器区。
片内RAM分区：

1. 工作寄存器区：00H\~1FH，分为4组R0\~R7，由PSW的RS1、RS0选择寄存器组；
2. 位寻址区：20H\~2FH，共128个可独立操作位；
3. 通用堆栈/数据区：30H\~FFH。
   片内额外集成1024字节XRAM，所有外设寄存器统一映射在XRAM地址空间。

## 3.特殊功能寄存器(SFR)

SFR分为外设XRAM寄存器、内核SFR寄存器两类；内核SFR占用80H\~FFH地址，和片内RAM地址重叠，但物理存储相互独立，访问指令区分。

### 3.1.内核特殊功能寄存器

| 寄存器  | 地址   | 功能说明         |
| ---- | ---- | ------------ |
| P0   | 0x80 | P0口数据寄存器     |
| SP   | 0x81 | 堆栈指针，上电默认07H |
| DPL  | 0x82 | DPTR低字节      |
| DPH  | 0x83 | DPTR高字节      |
| PCON | 0x87 | 电源控制寄存器      |
| PSEQ | 0x86 | PCON写保护配置寄存器 |
| VTOR | 0x88 | 中断向量偏移寄存器    |
| P1   | 0x90 | P1数据         |
| P2   | 0xA0 | P2数据         |
| IE   | 0xA8 | 全局中断使能       |
| IP   | 0xB8 | 中断优先级        |
| P4   | 0xC0 | P4数据         |
| PSW  | 0xD0 | 程序状态字        |
| ACC  | 0xE0 | 累加器          |
| B    | 0xF0 | 乘除辅助寄存器      |

#### PSW寄存器位定义

| bit7 | bit6 | bit5 | bit4 | bit3 | bit2 | bit1 | bit0 |
| ---- | ---- | ---- | ---- | ---- | ---- | ---- | ---- |
| CY   | AC   | F0   | RS1  | RS0  | OV   | F1   | P    |

\| CY：运算进位标志；AC：半进位；F0/F1：用户自定义标志；RS1/RS0：寄存器组选择；OV：溢出标志；P：累加器奇偶标志。 |

#### IE寄存器

bit7(EA)全局中断总开关，bit0\~bit5依次为EX0\~EX5六个内核中断单独使能位，未用到bit6。

#### IP寄存器

bit5\~bit0依次PX5\~PX0，对应6路内核中断优先级，1为高优先级，0低优先级；bit7/6无效。优先级规则：高优先级可抢占低优先级；同级不可嵌套；同级同时触发按PX0>PX1>…>PX5顺序响应。

#### PCON与PSEQ

PCON仅bit1(SLEEP)、bit0(IDLE)有效，不能同时置1；写入PCON前必须依次赋值PSEQ=0x0A、PSEQ=0x05解锁，调试模式无法软件外修改PSEQ、PCON。

#### VTOR

上电默认0x00，中断地址和标准8051一致；配置后所有中断入口高8位偏移，例VTOR=0x08，则INT0入口0x0803。

## 4.振荡器模块

### 4.1概述

芯片时钟来源三路：内部48M高速RC、内部800K低速RC、外接32.768K晶振；各路时钟经分频后供给CPU与各个外设。MDLCK寄存器统一管控各外设模块时钟开关。上电默认48M四分频=12MHz系统时钟；CPU最高主频为48M三分频16MHz，SYSDIV配置不能小于2。

### 4.2寄存器说明

#### CKDIVCR(0x50FF)

bit7\~6 SLOWDIV：内部800K分频，00÷1、01÷2、10÷4、11÷8；
bit5\~0 SYSDIV：48M分频系数，系统时钟=48M/(SYSDIV+1)，最低配置000010(3分频)。

#### MDLCKCR(0x51FF)

| bit | 标识      | 功能                          |
| --- | ------- | --------------------------- |
| 7   | 32KEN   | 1=Timer选用外接32.768K，0=内部800K |
| 6   | PWMEN   | PWM模块时钟开关                   |
| 5   | SPIEN   | SPI时钟开关                     |
| 4   | UARTEN  | UART时钟开关                    |
| 3   | IOCEN   | IO配置时钟开关，配置IO必须开启           |
| 2   | TIMEREN | 定时器总时钟                      |
| 1   | LEDEN   | LED驱动时钟                     |
| 0   | WDTEN   | 看门狗时钟                       |

> 使用任意外设前，必须打开MDLCK对应使能位。

## 5.MCU的特性

包含复位、中断、空闲/休眠低功耗、ESD防护、代码加密、在板串行烧录、在线调试七大功能。

### 5.1复位

复位相关寄存器：RSTSR(0x54FF)、LVDCR(0x55FF)、MISCR(0x56FF)。
复位源分为：POR上电复位、外部RESET引脚低电平复位、WDT二次溢出复位、LVD低压复位。

#### RSTSR

bit0(POR)上电复位标记、bit1(WDT)看门狗复位标记、bit2(LVD)低压复位、bit3(RESETPAD)外部引脚复位，标志位可软件清零。

#### MISCR

bit0 POREN：1禁止上电复位；bit2 FMULRDEN：FLASH多次读判决使能，D版本芯片固定关闭该功能；bit1 GFLTEN：OSC振荡输出滤波开启。

#### LVDCR

bit7 LVDSR低压告警标志；bit6\~4 FLTLEN毛刺滤波(1\~128us)；bit3\~2 VOL低压阈值(00=2.7V、01=2.9V、10=3.1V、11=3.3V)；bit1\~0 OPER低压处理方式：00暂停、01系统复位、10仅记录标志、11产生中断。

### 5.2中断

芯片所有外设中断汇总至6路内核中断，中断开启需要五级配置：外设自身中断使能→IRQnER分组使能→IP优先级配置→IE对应EXx开启→EA全局开启。

#### 六组内核中断分组

IRQ0(EX0)：LVD、WDT、T3INT0、XINT0；
IRQ1(EX1)：SensorADC、T3INT1、XINT1；
IRQ2(EX2)：RTC、T0、XINT2；
IRQ3(EX3)：UART、T1、XINT3；
IRQ4(EX4)：ADC、T2、XINT4；
IRQ5(EX5)：I2C主、I2C从、SPI、XINT5。

#### 外部中断配置寄存器

XINT01TRGR(INT0/1)、XINT23TRGR(INT2/3)、XINT45TRGR(INT4/5)，单路中断可选配置：高电平、低电平、上升沿、下降沿、双边沿、禁止触发；
IFLTCNTR：外部中断硬件滤波时钟个数，0\~256Tsys，上电默认16Tsys。

#### 中断标志规则

中断标志位硬件置1不受EA、局部中断开关影响，标志必须软件清零。

### 5.3休眠与空闲模式

#### 空闲IDLE

配置步骤：PSEQ=0x0A;PSEQ=0x05;PCON=0x01；CPU停止，全系统时钟正常运行；WDT/ADC/I2C主机停止工作；可通过全部中断唤醒。

#### 休眠SLEEP

配置步骤：PSEQ=0x0A;PSEQ=0x05;PCON=0x02；48M主时钟关闭，内部800K、外接32.768K继续运行；四种唤醒：外部INT0边沿唤醒、内部定时唤醒、RTC闹钟唤醒、指定触摸按键唤醒。

##### 休眠唤醒寄存器SLPWKCR(0x52FF)、SLPWKDR(0x53FF)

SLPWKCR.bit7 EDGESEL：INT0唤醒边沿选择(0下降沿、1上升沿)；bit6\~4 DRSEL用于分时读写24位自动唤醒定时/16位触控阈值；bit3 EINTWKEN外部0唤醒使能；bit2 SENSORWKEN按键唤醒；bit1 RTCWKEN闹钟唤醒；bit0 AUTOWKEN定时唤醒。
SLPWKDR配合DRSEL分段写入24位定时数值、双16位触控上下限。

### 5.4静电防护

芯片内置片上ESD检测硬件，遭遇静电冲击时芯片自动做防护处理，提升整机抗静电性能。

### 5.5代码保护

用户写入4字节加密秘钥后，程序FLASH加密，外部无法读出片内程序。

### 5.6在线串行编程

仅需要VDD、GND、TXD、RXD四根引脚即可实现在板烧写，可贴片后再烧录固件。

### 5.7在线调试

TXD、RXD复用调试口，连接串口即可实现在线调试；调试启用后P00/P01只能用作调试，不能做普通IO。

## 6.I/O端口

IO相关寄存器分为P0\~P4 SFR数据寄存器、PxyIOCFG功能寄存器、上下拉/开漏/驱动电流XRAM寄存器；上电所有IO默认输入、默认开启内部下拉。

### 6.1引脚配置为GPIO操作步骤

1.MDLCKCR.IOCEN置1，打开IO时钟；
2.对应PxyIOCFG=0x00，配置为GPIO；
3.写入Pn寄存器预设输出电平；
4.PnLDRVH/L配置输出驱动电流；
5.PnPURSEL配置上拉电阻档位；
6.PnPU选择是否开启上拉；
7.PnPD关闭/开启内部下拉；
8.PnOD配置开漏输出；
9.PnOE：1输入、0输出。

> 上下拉、开漏、方向配置互相独立；上电默认全部下拉，闲置IO建议关闭下拉降低功耗。

```c
//示例：P0.0输出，P1.0上拉输入
XBYTE[0x51FF] |= 0x08;
XBYTE[0x1000] = 0x00;
XBYTE[0x1008] = 0x00;
XBYTE[0x80] = 0x00;
XBYTE[0x90] = 0x00;
XBYTE[0x40FF] = 0x00;
XBYTE[0x42FF] = 0x00;
XBYTE[0x30FF] = 0x00;
XBYTE[0x32FF] = 0x00;
XBYTE[0x20FF] = 0x00;
XBYTE[0x21FF] = 0x01;
XBYTE[0x25FF] = 0x00;
XBYTE[0x26FF] = 0x00;
XBYTE[0x3AFF] = 0x00;
XBYTE[0x3BFF] = 0x00;
XBYTE[0x2AFF] &= ~0x01;
XBYTE[0x2BFF] |= 0x01;
```

### 6.2功能复用

PxyIOC低6位配置引脚复用功能：
000000=GPIO；000001=调试TX(P0.0专属)；000010=T1IO；000011=T3IO0；
000100=I2CSDA；000101=I2MCL；000110=I2MSDA；001000=触控K；001001=ADC AN；
01xxxx=SEG段脚；100001\~100100=GRID位脚；100101=T0O；100110=T2O；100111=TXD；
101000=PWM1；101001=PWM2；110000=RX(P0.1专属)；110001=I2SCL；110010=T3I1；
110011=RXD；110100\~110111=SPI从脚；111000\~111101=INT0\~INT5。

### 6.3方向选择

PnOE：对应位1=输入，0=输出；除T1/T3引脚外，其余复用功能自动配置IO方向，无需配置PnOE。Pn为端口数据寄存器，输出时写Pn，输入时读Pn。

### 6.4内部上拉

PnPU置1开启上拉；PnPURSEL组合配置阻值：00=88K、01=2.2K、10=4.5K、11=10K，默认88K。

### 6.5内部下拉

PnPD置1开启下拉，上电全部默认开启。

### 6.6开漏输出

PnOD置1开启开漏输出。

### 6.7驱动电流

PnLDRV配置：00=6mA、01=30mA、10=60mA、11=6mA；LED SEG/GRID引脚同样可配置电流实现灰度。

## 7.看门狗定时器(WDT)

寄存器：WDTCR(0x60FF)喂狗寄存器、WDTSETR(0x61FF)配置、WDTSR(0x62FF)中断标志。
WDT时钟来自48M分频后的系统时钟；第一次溢出置WDTINT标志产生中断，第二次溢出硬件复位；不可在中断服务函数内部喂狗。
\###7.1 WDT操作步骤
1.MDLCKCR.WDTEN=1开启WDT时钟；
2.WDTCR依次写入0x5A、0xA5预清零计数器；
3.WDTSETR配置CNTDIV分频、CNTLEN计数长度；
4.配置IRQ0ER.WDTINT、IE.EX0、EA开启溢出中断；
5.WDTSETR.CNTEN=1启动计数。
喂狗固定顺序：WDTCR=0x5A;WDTCR=0xA5，顺序颠倒无效。
\###7.2寄存器
WDTSETR.bit7 CNTEN计数器使能；bit6\~4 CNTDIV(1\~128分频)；bit3\~0 CNTLEN(1024\~16384)。
WDTS.bit0为WDT溢出标志，软件配合喂狗指令清零。

## 8.Timer0(8位定时器)

寄存器：T0CKCR(0x70FF)时钟、T0CR(0x71FF)模式、T0CMP0周期、T0CMP1占空、T0SR状态。
三种工作模式：定时、方波输出、PWM输出。
时钟可选48M/3高速、800K/32.768K低速。
\###8.1定时模式
周期=T0CMP0×计数时钟；T0CMP0=0按256计算；TEN关闭且T0SR.WREN=1才可改写T0CMP0。
\###8.2方波模式
输出引脚自动配置T0O，周期=2×T0CMP0×Tclk。
\###8.3PWM模式
T0CMP0=周期，T0CMP1=有效电平宽度，占空比=T0CMP1/T0CMP0；T0CMP1运行中可实时修改。

```c
//T0初始化示例
XBYTE[0x51FF] |= (1<<2);
XBYTE[0x70FF] = 0x11;
XBYTE[0x71FF] = 0x09;
XBYTE[0x72FF] = 120;
XBYTE[0x73FF] = 40;
```

\###8.2寄存器
T0CKCR.bit5 CLKEN时钟使能；bit4 CLKSEL时钟源；bit2\~0分频。
T0CR.bit3 INTEN中断使能；bit2 OUTSEL输出反转；bit1 MODE(0定时/1PWM)；bit0 TEN启停。
T0SR.bit1 WREN(T0CMP0可写)；bit0 INTSR中断标志。
\###8.3中断
计数器等于T0CMP0触发中断，空闲可唤醒，休眠无法唤醒。

## 9 8位定时器Timer

### 9.1 Timer1模块相关寄存器列表

| 寄存器名称 | 寄存器地址 | 地址类型 | 说明 | POR 复位值 |
| --- | --- | --- | --- |--- |
| T1CKCR | 0x78FF | XRAM | Timer1时钟控制寄存器 | 0000 0000 |
| T1CR | 0x79FF | XRAM | Timer1模式控制寄存器 | 0000 0000 |
| T1CMP | 0x7AFF | XRAM | Timer1比较寄存器 | 0000 0000 |
| T1SR | 0x7BFF | XRAM | Timer1状态寄存器 | 0000 0010 |

Timer1 模块是一个8 位定时器,具有如下特点:
8 位定时器
3 位预分频器
8 位周期与脉宽调制寄存器(T1CMP)
可编程外部时钟边沿选择
可编程内部高速或低速时钟源
可编程内部或外部时钟源

#### 9.1 Timer1支持的功能

Timer1 支持以下4 种功能:
间隔定时
方波输出
PWM 输出
外部事件计数

##### 间隔定时的操作步骤

配置Timer1 为间隔定时功能的操作步骤如下:

1. Timer 模块时钟使能:
   将MDLCKCR 寄存器的TIMEREN 位置1,使能Timer 模块时钟｡
2. Timer1 计数时钟参数配置:
   计数时钟选择:
   T1CKCR 寄存器的CLKSEL 位可以选择内部高速 OSC48M 的3 分频或者低速时钟作为Timer1 的时钟｡
   选择低速时钟时,还可以通过MDLCKCR 寄存器的 32KEN 位选择内部低速800K 或外部32.768K 晶振作为 Timer1 的时钟｡
   计数时钟分频配置:
   T1CKCR 寄存器的CLKDIV<3:0>位可以对选择的时 钟源进行分频｡此时不能配置为外部信号的有效沿选项｡
   计数时钟使能:
   将T1CKCR 寄存器的CLKEN 位置1,使能Timer1 计 数时钟｡
3. 工作模式选择:
   将T1CR 寄存器的MODE 位清0,工作模式选择为定 时器模式｡
4. 间隔时间配置:
   Timer1 的间隔定时时间是通过T1CMP 寄存器配置, 具体的间隔时间计算如下:
   \[间隔时间 =(T 1 C M P) x T i m e r 1 时钟周期]

Timer1 计数时钟周期
Timer1 计数时钟周期由选择的计数时钟决定,共 有三种时钟:
内部高速OSC48M 的3 分频
内部低速800K
外部32.768K晶振(必须确保外接了32.768K)
注: 1､T1CMP 在定时器模式下,Timer1 工作期间不能 修改,所以想要改变T1CMP 必须先把T1CR 寄存器 的TEN 位清零,等待T1SR 寄存器的WREN 位置1 后,再修改T1CMP｡
2､T1CMP 在PWM 模式下,Timer1 工作期间可以直 接修改｡
5\. 清Timer1 中断标志位
使能Timer1 中断前,先将T1SR 寄存器的INTSR 位清0,清Timer1 中断标志位｡
6\. 中断控制:
Timer1 只有一个中断,但是在间隔定时､方波输 出､PWM 输出､外部事件计数模式下,产生Timer1 中 断的条件不同｡在间隔定时､方波输出､外部事件计数 模式下,当Timer1 计数器值等于(T1CMP)时,会产生 Timer1 中断｡在PWM 输出模式下,当Timer1 计数器值 溢出,即等于0xFF 时,会产生Timer1 中断｡
T1CR 寄存器的INTEN 位是Timer1 中断使能位, T1SR 寄存器的INTSR 位是Timer1 中断标志位｡
7\. Timer1 使能:
将T1CR 寄存器的TEN 位置1,使能Timer1,Timer1 即开始工作｡

##### 方波输出的操作步骤

配置Timer1 为方波输出功能的操作步骤如下:

1. 端口配置:
   通过PxyIOCFG 寄存器将I/O 配置成定时器1 输出/ 输入功能,并且配置PnOE 寄存器,将定时器1 输出或 输入引脚方向置为输出｡(端口配置前应使能IOC 模块 时钟,即将MDLCKCR 寄存器的IOCEN 位置1)
2. Timer 模块时钟使能:
   将MDLCKCR 寄存器的TIMEREN 位置1,使能Timer 模块时钟｡
3. Timer1 计数时钟参数配置:
   计数时钟选择:
   T1CKCR 寄存器的CLKSEL 位可以选择内部高速 OSC48M 的3 分频或者低速时钟作为Timer1 的时钟｡
   选择低速时钟时,还可以通过MDLCKCR 寄存器的 32KEN 位选择内部低速800K 或外部32.768K 晶振作为 Timer1 的时钟｡
   计数时钟分频配置:
   T1CKCR 寄存器的CLKDIV<3:0>位可以对选择的时 钟源进行分频｡此时不能配置为外部信号的有效沿选项｡
   计数时钟使能:
   将T1CKCR 寄存器的CLKEN 位置1,使能Timer1 计 数时钟｡
4. Timer1 模式配置:
   输出翻转使能:
   T1CR 寄存器的INVEN 置1,使能当Timer1 计数器 值等于(T1CMP)时,输出翻转｡(不使能没有方波输出)
   输出初始状态选择:
   T1CR 寄存器的OUTINITV<1:0>位控制Timer1 输出 初始状态,只对方波输出有效,对PWM 输出无效｡
   工作模式选择:
   将T1CR 寄存器的MODE 位清0,工作模式选择为定 时器模式｡
5. 方波周期配置:
   方波周期是通过T1CMP 寄存器配置,Timer1 的方 波周期其实就是间隔时间的两倍,具体计算如下:
   \[\begin{aligned} & =2 \times\left(T 1 C M P) × T i m e 1 时钟周期 \ & =2 \times(T 1 C M P) × T i m e r 1 计数时钟分频 \end{aligned}]
   x Timer1 计数时钟周期
   Timer1 计数时钟周期由选择的计数时钟决定,共 有三种时钟:
   内部高速OSC48M 的3 分频
   内部低速800K
   外部32.768K晶振(必须确保外接了32.768K)
   注: 1､T1CMP 在定时器模式下,Timer1 工作期间不能 修改,所以想要改变T1CMP 必须先把T1CR 寄存器 的TEN 位清零,等待T1SR 寄存器的WREN 位置1 后,再修改T1CMP｡
   2､T1CMP 在PWM 模式下,Timer1 工作期间可以直 接修改｡
6. 清Timer1 中断标志位
   使能Timer1 中断前,先将T1SR 寄存器的INTSR 位清0,清Timer1 中断标志位｡
7. 中断控制
   Timer1 只有一个中断,但是在间隔定时､方波输 出､PWM 输出､外部事件计数模式下,产生Timer1 中 断的条件不同｡在间隔定时､方波输出､外部事件计数 模式下,当Timer1 计数器值等于(T1CMP)时,会产生 Timer1 中断｡在PWM 输出模式下,当Timer1 计数器值 溢出,即等于0xFF 时,会产生Timer1 中断｡
   T1CR 寄存器的INTEN 位是Timer1 中断使能位, T1SR 寄存器的INTSR 位是Timer1 中断标志位｡
8. Timer1 使能:
   将T1CR 寄存器的TEN 位置1,使能Timer1,Timer1 即开始工作｡
   **图9-1文字描述：Timer1间隔定时/方波时序图，纵向依次为计数时钟、计数器、比较值M=T1CMP、TEN引脚、INTSR标志、OUTINIT=01输出波形、OUTINIT=10输出波形；时序逻辑：①使能TEN，计数器从零开始逐次加1计数；②计数器等于M时，INTSR置1、输出电平翻转并触发中断；③软件清除中断标志；④TEN拉低后计数器清零停止；补充备注：T1CMP=0等效256，T1CMP=1无方波、无中断。**
   \[间隔时间 =(T 1 C M P) × Timer1 时钟周期]
   \[方波周期 =2 x 间隔时间]

##### PWM 输出的操作步骤

1. 端口配置:

   T1CKCR 寄存器的CLKDIV<3:0>位可以对选择的时 钟源进行分频｡此时不能配置为外部信号的有效沿选项｡

4. Timer1 模式配置:

   工作模式选择:
   将T1CR 寄存器的MODE 位置1,工作模式选择为PWM 模式｡
5. PWM 占空比配置:
   Timer1 的PWM 占空比配置与Timer0 有所区别,只 通过T1CMP 配置,具体计算如下:
   \[PWM 周期 =(0 xFF+1) x Timer1 时钟周期 \frac{1}{2}=(0 xFF+1) x Timer1 计数时钟分频 x]
   Timer1 计数时钟周期
   Timer1 计数时钟周期由选择的计数时钟决定,共 有三种时钟:
   内部高速OSC48M 的3 分频
   内部低速800K

   使能Timer1 中断前,先将T1SR 寄存器的INTSR 位清0,清Timer1 中断标志位｡
7. 中断控制:

8. Timer1 使能:
   将T1CR 寄存器的TEN 位置1,使能Timer1,Timer1 即开始工作｡
   **图9-2文字描述：Timer1 PWM时序，纵向：计数时钟、计数器、比较值N=T1CMP、TEN、INTSR、OUTSEL=0输出、OUTSEL=1输出；时序：①TEN开启计数器从0递增；②计数到N，输出电平翻转；③计数到0xFF溢出，INTSR置1、输出再次翻转；④软件清中断；⑤TEN关闭计数器清零；备注：T1CMP=0等效256，修改T1C下个周期生效。**

   \[PWM 占空比 =(T 1 C M P):(0 x F F+1)]
   注:1、当 (T 1 CMP=0) 时,按T1CMP=256计算,此时产生方波。
   2､若需要产生可变占空比的PWM 波形,可以修改T1CMP 寄存器｡软件修改T1CMP 寄存器后,在下个PWM 周期才会生效｡

##### 外部事件计数的操作步骤

配置Timer1 为外部事件计数功能的操作步骤如下:

1. 端口配置:
   通过PxyIOCFG 寄存器将I/O 配置成定时器1 输出/ 输入功能,并且配置PnOE 寄存器,将定时器1 输出或 输入引脚方向置为输入｡(端口配置前应使能IOC 模块 时钟,即将MDLCKCR 寄存器的IOCEN 位置1)
2. Timer 模块时钟使能:
   将MDLCKCR 寄存器的TIMEREN 位置1,使能Timer 模块时钟｡
3. Timer1 计数时钟参数配置:
   外部信号的有效沿选择:
   T1CKCR 寄存器的CLKDIV<3:0>位可以对外部信号 的有效沿进行选择,有上升沿､下降沿､双边沿｡此时 不能配置为计数时钟分频选项｡
   计数时钟使能:
   将T1CKCR 寄存器的CLKEN 位置1,使能Timer1 计 数时钟｡
4. 工作模式选择:
   将T1CR 寄存器的MODE 位清0,工作模式选择为定 时器模式｡
5. 匹配值设置:
   在外部事件计数模式下,Timer1 计数器在定时器1 输入引脚的每个有效沿都会加1,当计数器值等于 (T1CMP)时,会产生中断｡另外T1CMP ≠ 1,T1CMP = 0 时,按256 计算｡
   注: 1､T1CMP 在定时器模式下,Timer1 工作期间不能 修改,所以想要改变T1CMP 必须先把T1CR 寄存器 的TEN 位清零,等待T1SR 寄存器的WREN 位置1 后,再修改T1CMP｡
   2､T1CMP 在PWM 模式下,Timer1 工作期间可以直 接修改｡
6. 清Timer1 中断标志位
   使能Timer1 中断前,先将T1SR 寄存器的INTSR 位清0,清Timer1 中断标志位｡
7. 中断控制:
   Timer1 只有一个中断,但是在间隔定时､方波输 出､PWM 输出､外部事件计数模式下,产生Timer1 中 断的条件不同｡在间隔定时､方波输出､外部事件计数 模式下,当Timer1 计数器值等于(T1CMP)时,会产生 Timer1 中断｡在PWM 输出模式下,当Timer1 计数器值 溢出,即等于0xFF 时,会产生Timer1 中断｡
   T1CR 寄存器的INTEN 位是Timer1 中断使能位, T1SR 寄存器的INTSR 位是Timer1 中断标志位｡
8. Timer1 使能:
   将T1CR 寄存器的TEN 位置1,使能Timer1,Timer1 即开始工作｡
   **图9-3文字描述：外部事件计数时序，纵向：输入引脚(CLKDIV=1001上升沿)、计数器、M=T1CMP、TEN、INTSR；①TEN开启，输入有效沿计数器+1；②计数等于M，INTSR置1触发中断；③软件清中断；④TEN关闭计数器清零；备注：T1CMP=0→256，输入信号频率<计数时钟1/2，T1CMP=1无中断。**

### 9.2 Timer1寄存器的定义

#### 寄存器9-1 T1CKCR(0x78FF)

| bit7 | bit6 | bit5 | bit4 | bit3 | bit2 | bit1 | bit0 |
| --- | --- | --- | --- | --- | --- | --- |--- |
| - | - | CLKEN | CLKSEL | CLKD3 | CLKD2 | CLKD1 | CLKD0 |

| U-0 | U-0 | R/W-0 | R/W-0 | R/W-0 | R/W-0 | R/W-0 | R/W-0 |

> 图注：R可读、W可写、U未实现读0；-n上电复位值
> bit7\~6：未实现，读0
> bit5 CLKEN：0关闭计数时钟，1开启
> bit4 CLKSEL：0=48M三分频，1=低速(800K/32.768K由32KEN选择)
> bit3\~0 CLKDIV：
> 0000=2分频、0001=4、0010=8、0011=16、0100=32、0101=64、0110=128、0111=256
> 1001上升沿、1010下降沿、1011双边沿；其余编码=1分频

#### 寄存器9-2 T1CR(0x79FF)

| bit7 | bit6 | bit5 | bit4 | bit3 | bit2 | bit1 | bit0 |
| --- | --- | --- | --- | --- | --- | --- |--- |
| - | INVEN | OUTINIT1 | OUTINIT0 | INTEN | OUTSEL | MODE | TEN |
| U-0 | R/W-0 | R/W-0 | R/W-0 | R/W-0 | R/W-0 | R/W-0 | R/W-0 |

bit7：保留读0
bit6 INVEN：匹配是否翻转输出；bit5\~4输出初始：00/11保持、01低、10高
bit3 INTEN：中断开关；bit2 OUTSEL(PWM极性)；bit1 MODE(0定时/1PWM)；bit0 TEN启停

#### 寄存器9-3 T1CMP(0x7AFF)

8位全R/W-0；
说明：定时模式运行不可改，需TEN=0+WREN=1；PWM任意修改。

```c
//范例7-1：定时器模式修改T1C
T1CR_TEN = 0x00;
while(0x00 == T1SR_WREN){;}
T1CMP = 0x55;
//其他处理
T1CR_TEN = 0x01;
```

#### 寄存器9-4 T1SR(0x7BFF)

| bit7\~2 | bit1(WREN) | bit0(INTSR) |
| ------- | ---------- | ----------- |
| U-0     | R-1        | R/W-0       |

WREN=1可改写T1CMP；INTSR软件清零。

### 9.3 Timer1中断

产生Timer1 中断有以下三种方式:
间隔定时/方波输出/外部事件计数模式下,当 Timer1 计数器值等于(T1CMP)时;
PWM 模式下,当Timer1 计数器值溢出,即等于0xFF 时｡
只要有Timer1 中断条件产生,不论是否使能Timer1 中 断,T1SR 寄存器的INTSR 位都会置1, INTSR 位必须在软件中清零｡Timer1 的中断使能位是T1CR 寄存器的INTEN 位｡
注: 在休眠模式下,Timer1 中断无法唤醒处理器｡ 在空闲模式下,Timer1 中断可以唤醒 处理器｡

##### 间隔定时步骤

1. Timer模块时钟：MDLCKCR.TIMEREN=1
2. T2CK配置：CLKSEL选高速/低速，低速由32KEN选800K/32.768K；CLKDIV分频，CLKEN=1
3. T2CR.MODE可选0/1都支持定时
4. 间隔时间 = (T2CMP0H:T2CMP0L) × 时钟周期；三种时钟同上；

> T2CMP0运行不可改，TEN=0，WREN=1才可改写

##### 方波输出

1. IO配置PxyIOCFG设T2输出，IOCEN提前开启，自动设输出，无需PnOE
2. 时钟同定时；T2CR.MODE=0，OUTSEL控制翻转；方波周期=2×间隔时间

3. 清标志、开中断、TEN=1

##### PWM输出

1. IO自动输出；MODE=1；PWM周期=(T2CMP0组合)×时钟，占空=(T2CMP1):(T2CMP0)；T2CMP1运行可实时修改，下个周期生效；T2CMP0停机改。
   \[PWM 周期 = (T2CMP0H:T2CMP0L) × Timer2时钟周期]
   \[PWM占空比=(T2CMP1H:T2CMP1L):(T2CMP0H:T2CMP0L)]
   T2CMP0全0等效65536。

|USETR|0xD1FF|XRAM|UART数据格式控制寄存器|0000 1000|
|UBPRL|0xD2FF|XRAM|UART波特率配置寄存器低8位|1111 1111|

通用异步收发器(UART)模块是一个串行I/O通信外设，它可被配置为能与CRT终端和个人计算机等外设通信的全双工异步系统。

UART模块包含如下功能：
- 全双工异步发送和接收
- 可将字符长度编程为5/6/7/8位
- 输入缓冲器溢出错误检测
- 停止位错误检测
- 奇偶校验位错误检测
- 自动波特率检测和校准

每个发送的字符包括一个起始位，后面跟有5/6/7/8个数据位，若有奇偶校验位就再加上1位校验位，以及1个或多个终止字符发送的停止位。最常用的数据为8位数据位不做奇偶校验。每个发送位的持续时间为1/(波特率)。片上专用12位波特率发生器可用于通过振荡器产生标准波特频率(见寄存器UBPRH: UBPRL)。

UART首先发送和接收LSB。UART的发送器和接收器在功能上是相互独立的，但采用相同的数据格式和波特率。

## 16.1. UART的操作步骤
配置UART的操作步骤如下：
### 1. 端口配置
通过PxyIOCFG寄存器将I/O配置成UART输出/输入功能时，会自动将引脚方向配置为相应的方向。不需要配置对应的PnOE寄存器，选择UART输出/输入引脚方向为输出/输入。(端口配置前应使能IOC模块时钟，即将MDLCKCR寄存器的IOCEN位置1)

### 2. UART模块时钟使能
在操作UART模块相关寄存器前，必须首先使能UART模块时钟，即将MDLCKCR寄存器的UARTEN位置1。

### 3. UART模块参数配置：
#### ● 数据格式配置
USETR寄存器可以配置通讯数据的格式，包括校验位、数据位、停止位。

#### ● 波特率配置
波特率有两种配置方式(通过UCR寄存器的BPRREC位选择)：
1) 自动波特率校准（上电第一次校准有效）
采用自校准方式。上位机必须先发送0x11，芯片分析上位机的波特率，在以后的通讯过程中，都采用和上位机一致的波特率通讯。此种情况对芯片内部时钟精准度无特别要求，但上位机必须发送特定数据。

2) 12位波特率配置寄存器
通讯的波特率依据12位波特率配置寄存器UBPRH:UBPRL的设置确定。在时钟比较准确的情况下，可以采用这种方式。
UART的时钟源是OSC48M/3的时钟(16M)，UBPRL为UBPR值的低8位，UBPRH为UBPR的高4位。UBPR是根据波特率推算的配置数：
例如：波特率bpr是115200，计算UBPR如下：
\[
\begin{aligned}
UBPR &= F / bpr - 1；\ (F的单位是Hz) \\
&= 16 * 1000000 / 115200 - 1 \\
&= 137(0x89)
\end{aligned}
\]
所以UBPRH = 0x00，UBPRL = 0x89。

#### ● 中断控制
UIER寄存器的TXIE、RXIE位分别是UART发送完1个BYTE数据中断使能位、接收完1个BYTE数据中断使能位。对应的中断标志位是USR寄存器的TXDONE、RXREADY位。

不论是否使能发送完成、接收完成中断，当接收到1个BYTE数据时，RXREADY都会置1；当发送完1个BYTE数据时，TXDONE都会置1。

USR寄存器还包括UART的奇偶校验位错误标志位、停止位错误标志位、接收溢出错误标志位，这几种情况不会产生中断，当出现相应错误时标志位就会置1。

### 4. 接收、发送使能
当端口、数据格式、波特率都配置好后，可以将UCR寄存器的RXEN位置1使能数据接收，TXEN位置1使能数据发送。数据的发送和接收功能是相互独立的，不会受另一方的影响。

## 16.2. 发送数据
UTXR为发送数据寄存器，向UTXR寄存器写入一个字符，即启动发送。只有当USR寄存器的TXDONE位 = 1时，才能将待发送的数据写入到UTXR寄存器中，否则写入的数据会丢失。TXDONE是只读位，不能由软件清0。当TXDONE = 1时，向UTXR寄存器写入一个字符后，TXDONE会由硬件自动清0。

如果要在发送数据时使用中断，只在有待发送数据时，才将TXIE位置1。当将待发送的最后一个字符写入UTXR后，软件必须要将TXIE位清0。

## 16.3. 接收数据
接收器在第一个位的下降沿开始接收字符。第一个位，通常称为起始位，始终为0。如果起始位校验出错，则放弃接收该字符，只有在接收到正确的起始位，才会继续采样，最后一个位为停止位，总是为1。如果采样停止位时，采样到0，则会将停止位错误检测位置1。

当完成所有数据的采样并将其全部移入接收缓冲寄存器时，USR寄存器的RXREADY位会被置1，通过读URXR寄存器将缓冲寄存器的数据读出后，RXREADY位会自动清0。如果在接收到1 Byte数据后，软件未及时读URXR，当接收到新的数据时，就会发生接收溢出错误。(新的数据会丢失)

```c
//UART基础初始化示例
void UART_Init(void)
{
    XBYTE[0x51FF] |= (1<<4);    //MDLCKCR.UARTEN=1，开启UART模块时钟
    //端口配置省略：提前配置TXD/RXD引脚复用UART功能，IOCEN已开启
    XBYTE[0xD1FF] = 0x08;       //USETR：8数据位，1停止位，无校验
    //115200波特率配置，UBPR=137=0x89
    XBYTE[0xD2FF] = 0x89;       //UBPRL=0x89
    XBYTE[0xD3FF] = 0x00;       //UBPRH=0x00
    XBYTE[0xD0FF] = 0x03;       //UCR：RXEN=1、TXEN=1，关闭自动波特率校准
    XBYTE[0xD4FF] = 0x40;       //UIER：使能接收中断，关闭发送中断
}

//单字节发送函数
void UART_SendByte(unsigned char dat)
{
    while((XBYTE[0xD7FF] & (1<<4)) == 0); //等待TXDONE=1
    XBYTE[0xD5FF] = dat;
}

//单字节查询接收
unsigned char UART_RecvByte(void)
{
    while((XBYTE[0xD7FF] & (1<<3)) == 0); //等待RXREADY=1
    return XBYTE[0xD6FF];
}
```

SPI 模块允许同时同步发送和接收8 位数据｡支持SPI 的所有4 种时序,使用以下4 个引脚来完成通信:
SPI 数据输出
SPI 数据输入
SPI 时钟信号
SPI 片选信号
SPI 模块发送和接收都有4 Byte 深度的数据缓冲器,在 启动一组数据的传送前,必须软件写SPICR寄存器的FIFOCLR 位为1 清空数据缓冲器,防止FIFO 中存在以前未传送完的数据｡FIFOCLR 位为只写位,读为0｡

## 17.1 SPI 的操作步骤
配置SPI 的操作步骤如下:
1. 端口配置
SPI 有主控和从动两种工作模式｡所以存在两组端 口配置方式｡
SPI 配置为主控模式时,可通过PxyIOCFG 寄存器 将I/O 配置成SPIM 数据输入､SPIM 数据输出､SPIM 片选输出､SPIM 时钟输出功能｡
SPI 配置为从动模式时,可通过PxyIOCFG 寄存器 将I/O 配置SPIS 数据输入､SPIS 数据输出､SPIS 片选 输入､SPIS 时钟输入功能｡
只要配置好后,会自动将引脚方向配置为相应的方 向｡不需要配置对应的PnOE 寄存器,选择SPI 引脚方 向为输出/输入｡(端口配置前应使能IOC 模块时钟,即 将MDLCKCR 寄存器的IOCEN 位置1)

2. SPI 模块时钟使能
在操作SPI 模块相关寄存器前,必须首先使能SPI 模块时钟,即将MDLCKCR 寄存器的SPIEN 位置1｡

3. SPI 模块参数配置:
### 主从模式选择
SPI 有主控和从动两种工作模式｡通过SPICR 寄存 器的MSSEL 位控制｡ 必需注意的是SPICR 的MSSEL 位 与其他位不能同时配置,必需先写SPICR 的MSSEL 位配 置好模式后,才能配置其他位｡

### 时钟速率(仅限主控模式)
时钟线是由主器件控制的,所以在主控模式下,时 钟速率的配置才有效｡当作为主控模式时,SPIBPR 寄 存器可以控制SPI 通讯的波特率｡具体计算方法如下:
\[SPI 每个 bit 的时间 =2 *(SPIBPR+1) * Tspi;\]
其中Tspi 为SPI 的时钟周期,SPI 时钟是OSC48M 的3 分频(也就是16Mhz)｡

### 片选信号产生方式选择
当SPI 为主控模式时,可以通过SPICR 寄存器的 CSMODE 位选择片选信号产生方式,有两种方式:
1) 硬件产生片选信号,此时通讯过程中,硬件 自动产生拉高､拉低片选信号｡
2) 软件产生片选信号,此时通过SPISFMR 寄存 器控制片选信号拉高､拉低｡必须在时钟信 号发送前将片选信号拉低,数据传送完成后 再将片选信号拉高｡若传送中片选拉高则会 出错｡

### 发送接收位顺序配置
SPICR 寄存器的DORD 位清0,选择LSB 方式,发送 时,先发送字节低位,接收时,先接收到的为字节低位｡
SPICR 寄存器的DORD 位置1,选择MSB 方式,发送 时,先发送字节高位,接收时,先接收到的为字节高位｡

### 时钟极性､时钟相位配置
SPI 有4 中不同的时序,通过SPICR 寄存器的CPHA､ CPOL 位,可配置成不同的通讯时序,CPHA 位配置时钟 相位(采样的时间,上升沿或下降沿),CPOL 位配 置时钟极性(时钟线的空闲状态)。
**图17-1文字描述：CPHA=0时序图，纵向依次为时钟编号、SCK(CPOL=0)、SCK(CPOL=1)、DORD=0 MOSI、DORD=0 MISO、DORD=1 MOSI、DORD=1 MISO、CS；红线标注各采样时刻，低位优先与高位优先的数据采样位置区分。**
**图17-2文字描述：CPHA=1时序图，纵向依次为时钟序号、SC(CPOL=0)、SC(CPOL=1)、DORD=0 MOSI、DORD=0 MISO、DORD=1 MOSI、DORD=1 MISO、CS；红线标注对应采样点位。**

4. 中断控制
SPIIER 寄存器是SPI 中断使寄存器,有 SPIIER<4:0>5 个位与SPISR<4:0>一一对应,分别是发 送FIFO 空中断､发送FIFO 满中断､接收FIFO 非空中 断､接收FIFO 半满中断､接收FIFO 满中断｡SPISR<4:0> 就是相应的中断标志位｡不论中断是否使能,只要产生 中断条件,相应的标志位就会置1｡
SPISR 寄存器还有RXFOV(接收FIFO 溢出标志位)､ FWRSR(发送FIFO 写状态位)､TRANS(SPI Master 数据 传送启动位)｡这些都只是标志位,不会产生中断｡详 细请查看寄存器介绍｡

5. 接收发送使能
当参数都配置好以后,可以将SPICR 寄存器的RXEN 位置1 使能数据接收,TXEN 置1 使能数据发送｡数据 的发送和接收功能是相互独立的,不会受另一方的影响｡

## 17.2. 主控模式
主器件控制时钟线,因此可以随时启动数据传输｡主器 件根据软件协议确定从器件何时传输数据｡
1)数据发送:
在主控模式下,软件通过写SPIDR 寄存器,将数据写入 到发送FIFO 中｡最多可写4 字节数据,FIFO 已满将不能再写入数据｡软件应先查询发送FIFO 满标志位为0,再写SPIDR｡
数据写入发送FIFO 后,将SPISR 寄存器的TRANS 位置1 即启动1Byte 的数据传送,软件查询TRANS 位为0 时,即表示Byte 传输结束｡必须等待TRANS 位为0 后,才能再次将 TRANS 位置1 启动下次Byte 传输｡每完成1Byte 数据传送, 发送FIFO 的数据都会减少1Byte,SPISR 寄存器的相应标志 位也会做出改变｡

2)数据接收:
在主控模式下,接收的数据先存入到接收FIFO 中,深 度也为4Byte,软件读SPIDR 寄存器,可将接收FIFO 中的数 据取出｡
每读一次SPIDR 寄存器,接收FIFO 中缓存的数据都会 减少1Byte,SPISR 寄存器的相应标志位也会做出改变｡软 件上先查询接收FIFO 非空标志位为1,再读取SPIDR｡如果 接收FIFO 已满,SPI 仍继续接收数据,则会溢出｡

3)时钟速度:
理论上SPI 的时钟速度可达到1.14mhz,但是需要通过 相关寄存器来配置｡

## 17.3. 从动模式
在从动模式下,当时钟引脚上出现外部时钟脉冲时,即 启动数据发送和接收｡当最后一位数据被锁存后,SPISR 寄 存器的RXFNOTEMPTY､RXFHALF､RXFFULL 位会产生相应变化｡
1)数据发送和接收:
在从动模式下,发送接收数据与主控模式相同,区别是, 从动模式不能控制何时通讯,只能被动的响应主器件的数据 传送｡
2)片选信号:
从动模式下,必须接收到片选信号,才会响应数据传送｡ 就是说只有片选引脚为低电平时,才能发送和接收数据｡
注: 软件仅在SPISR 寄存器的FWRSR 位为0 时,才可 以写SPIDR 寄存器将待发送数据写入发送FIFO 中｡

## 17.4. SPI 寄存器的定义

## 18.1. I2C Slave(从动模块)
I 2C Slave 有4 种工作方式:
 I2C Slave 硬件快速模式(只读)
 I2C Slave 硬件标准模式(只读)
 I2C Slave 软件快速模式(只读)
 I2C Slave 软件标准模式(读写)
注:以上标准模式和快速模式是指I2C 的工作方式(请 参考图18-1,18-2,18-3,18-4,18-5),并非指I2C 的传输速率级别｡
在这4 种模式下的通讯时序有所不同｡硬件快速､硬件 标准､软件快速这三种模式下,主机只能从从机读数据而不 能对从机写数据｡只有软件标准模式下,主机从可以对从机 进行读写｡

### I2C Slave 的操作步骤
配置I 2C Slave 的操作步骤如下:
1. 端口配置
I 2C Slave 时钟线后,会自动将引脚方向配置为相应的 方向｡不需要配置对应的PnOE 寄存器,选择I 2C 引脚方 向为输出/输入｡(端口配置前应使能IOC 模块时 钟,即将MDLCKCR 寄存器的IOCEN 位置1)
PnPU 寄存器可以使能对应时钟､数据线的内部上 拉电阻,PnPURSELRH:PnPURSELRL 寄存器可以配置内 部上拉电阻的大小｡

2. I2C Slave 模块参数配置:
#### 设备地址配置
主机发送开始信号后,紧接7 位地址+1 位读/写位, 这7位地址数据必须与I2CSADDR寄存器的<6:0>位匹配, 从机才会做出应答｡通过配置I2CSADDR 寄存器为不同 的值,可以实现多机通讯(一主多从)｡I2CSADDR 的第7 位是记录主机读写状态的位,0 表示写,1 表示读｡

#### 硬件地址配置(在硬件标准模式下)
硬件地址配置只在硬件标准模式下有作用｡只有在 硬件标准模式下,当发送的设备地址匹配后,需要接着 发送8 位硬件地址｡当这8 位硬件地址数据必须与 I2CSHADR 寄存器匹配时,从机才会做出应答｡

#### 待发送数据配置
主机读从机数据时,从机必须先把待发送数据写入 内部数据寄存器中｡通过I2CSDRSEL和I2CSDR寄存器, 将待发送数据写入到内部数据寄存器中｡
1) 硬件模式下,内部数据寄存器为4 个字节:
配置 \(I 2 CSDRSEL =00\) ,将数据字节0写到I2CSDR寄 存器;
配置 \(I 2 CSDRSEL =01\) ,将数据字节1写到I2CSDR寄 存器;
配置 \(I 2 CSDRSEL =10\) ,将数据字节2写到I2CSDR寄 存器;
配置I2CSDRSEL=11,将数据字节3 写到I2CSDR 寄 存器｡
2) 软件模式下,内部数据寄存器为2 个字节:
配置 \(I 2 CSDRSEL =00\) ,将数据字节0写到I2CSDR寄 存器;
配置 \(I 2 CSDRSEL =01\) ,将数据字节1写到I2CSDR寄 存器｡

#### 工作模式配置
I2C Slave 有4 种工作模式,通过I2CSCR 寄存器的 MODE<1:0>位可以配置I2C 的工作模式:
\[MODE < 1:0>=00; 软件标准模式\]
\[MODE < 1:0>=01; 软件快速模式\]
\[MODE < 1:0>=10; 硬件标准模式\]
\[MODE < 1:0>=11; 硬件快速模式\]

3. 清I2C Slave 中断标志位
使能I2C Slave 中断前,先将I2CSSR 寄存器清0,清I2C Slave 相关中断标志位｡

4. 中断控制
I2CSCR 寄存器的INTEN 位是I2C Slave 中断使能位, 有INTEN<3:0>4个位与I2CSSR寄存器的<3:0>位一一对 应,分别是I2C Slave 接收到地址中断､I2C Slave 接收到字节数据中断､I2C Slave 发送完字节数据中断､I2C Slave 接收到设备地址中断(只在软件快速模式下产生)｡I2CSSR<3:0>就是相应的中断标志位｡不论中断 是否使能,只要产生中断条件,相应的标志位就会置1｡ 需要注意的是,在不同模式下,产生不同的中断:
1)硬件快速模式下,不产生中断;
2)硬件标准模式下,只产生I2C Slave 接收到地 址中断,此时的地址是硬件地址,保存在I2CSRADR 寄 存器中;
3)软件快速模式下,只产生I2C Slave 接收到设备地址中断、I2C Slave 发送完字节数据中断;
4)软件标准模式下,只产生I2C Slave 接收到地 址中断､I2C Slave 接收到字节数据中断､I2C Slave 发送完字节数据中断｡

5. 使能响应主机读写数据
I2CSCR 寄存器的ACKEN 位是使能响应主机读写数 据位,只有将ACKEN 置为1,从机才会响应主机的命令; 否则就算时序正确,从机也不会回应主机｡

### I2C Slave 硬件快速模式(只读)
软件操作步骤:
1､I2C Slave 配置成硬件快速模式,使能响应主机读写;
2､通过I2CSDRSEL 寄存器和I2CSDR 寄存器,将预发送 数据写入到内部数据寄存器中,最多可写入4 字节数据;

### I2C Slave 硬件标准模式(只读)

2､配置I2CSHADR 寄存器;

### I2C Slave 软件快速模式(只读)
软件操作步骤:
1､I2C Slave 配置成软件快速模式,使能响应主机读写;
2､使能接收到设备地址中断;
3､通过I2CSDRSEL 寄存器和I2CSDR 寄存器,将预发送 数据写入到内部数据寄存器中｡最多可以写入2 字节数据;
4､I2C 主机发送匹配的设备地址和读标志,即可从Slave 读取最多2 个字节数据;
5､从机中断触发后,通过查询发送完字节数据中断标 志位,通过I2CSDRSEL 寄存器和I2CSDR 寄存器,将后续待 发送数据写入到内部数据寄存器中｡

### I2C Slave 软件标准模式(读写)
软件操作步骤:
1､I2C Slave 配置成软件标准模式,使能响应主机读写;
2､使能接收到地址中断;
3､通过I2CSDRSEL 寄存器和I2CSDR 寄存器,将预发送 数据写入到内部数据寄存器中｡最多可以写入2 字节数据;
4､若主机执行写操作,I 2C 主机发送匹配的设备地址和 写标志,再送入内存地址,再紧跟待写入数据｡从机中断触 发后,根据接收到的内存地址的值以及接收数据,将数据存 入到对应的地址中;
6､若主机执行读操作,I 2C 主机发送匹配的设备地址和 写标志,再送入内存地址｡然后主机再次发送匹配的设备地 址和读标志｡从机中断触发后,通过查询发送完字节中 断标志位,根据接收到的内存地址的值,调配相应的数据, 通过I2CSDRSEL 寄存器和I2CSDR 寄存器,将后续待发送数 据写入到内部数据寄存器中｡

## 18.2. I2C Master(主控模块)
所有串行时钟脉冲和启动/停止条件均由主器件产生｡ 停止条件或重复启动条件能结束传输｡因为重复启动条件也 就是下一次串行传输的开始,因此不会释放I2C 总线｡
在主控发送模式下,串行数据通过数据线输出,而串行 时钟由时钟线输出｡发送的第一个字节包括7 位从器件设备 地址(7 位)和R/W 位｡在发送模式下,R/W 位将是逻辑0｡ 因此,发送的第一个字节是一个7 位从器件地址,后面 跟0 表示发送｡串行数据每次发送8 位｡每发送一个字节, 会收到一个应答位｡启动和停止条件的输出表明串行传输 的开始和结束｡
在主控接收模式下,串行数据通过数据线接收,而串行 时钟由时钟线输出｡发送的第一个字节包括发送器件的设备 地址(7 位)和R/W 位｡在接收模式下,R/W 位将是逻辑1｡ 因此,发送的第一个字节是一个7 位从器件设备地址,后面 跟1 表示接收｡每次接收8 位串行数据｡每接收到一个字节, 都会发送一个应答位｡启动和停止条件的输出表明串行传输 的开始和结束｡
在I2C 主控模式下,可以通过I2CMPRERH:I2CMPRERL 寄 存器配置I2C 通讯的波特率｡

### I2C Master 的操作步骤
1. 端口配置
当IO 复用功能寄存器配置为I2C Master 数据线､I2C Master 时钟线后,会自动将引脚方向配置为相应的 方向｡不需要配置对应的PnOE 寄存器,选择I2C Master 引脚方向为输出/输入｡(端口配置前应使能IOC 模块时 钟,即将MDLCKCR 寄存器的IOCEN 位置1)
PnPU 寄存器可以使能对应的时钟数据线的内部上 拉电阻,PnPURSELRH:PnPURSELRL 可以配置内部上拉 电阻的大小｡

2. 波特率配置
I2C Master 模块的时钟是由OSC48M 经过MDLCKCR 寄存器的SYSDIV<5:0>位分频后的时钟｡通过配置 I2CMPRERH: I2CMPRERL 寄存器的值,可以调整I2C Master 的通讯波特率｡
\[I2C Master 波特率 = OSC48M / (SYSDIV+1) / (I2CMPRERH:I2CMPRERL + 1) × 5;\]
\[I2CMPRERH: I2CMPRERL =48 MHz / 3 /(5 * 100 KHz)-1 = 31 = 0x1F;\]

3. 清I2C Master 中断标志位
使能I2C Master 中断前,先将I2CMCTR 寄存器的 CLRIF 位置1,清I2C Master 中断标志位｡

4. 中断控制
I2C Master 只有一个中断,若中断使能,每完成8 位数据传输或者当总线仲裁失败就会产生中断｡中断使 能位是I2CMCR 寄存器的IE 位,I2CMSR 寄存器的IF 位 是对应的中断标志位｡不论中断是否使能,只要产生中 断条件,相应的标志位就会置1｡
I2CMSR 寄存器除了IF 中断标志位,还有许多标志 位,这些只是标志位,不会产生中断｡详细请查看寄存 器定义｡

5. 主控模式使能
将I2CMCR 寄存器的I2CMEN 位置1,使能I2C Master 模块功能｡使能I2C Master 后,才能进行数据传输｡

6. 传输控制
一旦使能主控模式,用户可以进行以下操作:
 发出一个启动条件
 产生一个停止条件
 开始数据/地址的发送
 开始接收数据
 在接收到数据字节后产生应答条件
这些操作都是通过I2CMCTR 寄存器来控制的,但是 需要注意的是启动条件､停止条件和应答条件都不是单 独控制,必须和8 位数据/地址的传输组合在一起｡
例如发出一个启动条件:
1) 因为启动条件后面必须接一个7 位设备地址 加一位R/W 位,所以先把传送的7 位设备地址+1 位R/W 位的数据写入I2CMDR｡
2) 把I2CMCTR 寄存器的START 位､WR 位同时置1｡
3) START、WR 置1 后自动发出起始+发送字节,传输完成自动清零。

```c
//例1:向从机写1个字节 从机地址:0x4E 字节数据:0x65
I2CMDR = 0x9C;
I2CMCTR = 0x90;
while(0x02 == I2CMSR_TIP) {;}
if(0x00 == I2CMSR_RXACK) {
    I2CMDR = 0x65;
    I2CMCTR = 0x50;
    while(0x02 == I2CMSR_TIP);
}

//例2:读取从机1个字节 从机地址:0x52 读地址0x20
I2CMDR = 0xA4;
I2CMCTR = 0x90;
while(0x02 == I2CMSR_TIP);
if(0 == I2CMSR_RXACK){
    I2CMDR = 0x20;
    I2CMCTR = 0x10;
    while(0x02 == I2CMSR_TIP);
    if(0 == I2CMSR_RXACK){
        I2CMDR = 0xA5;
        I2CMCTR = 0x90;
        while(0x02 == I2CMSR_TIP);
        if(0 == I2CMSR_RXACK){
            I2CMCTR = 0x62;
            while(0x02 == I2CMSR_TIP);
            dat = I2CMDR;
        }
    }
}
```

CBM73xx 系列芯片的Flash 存储器包括Main 区和NVR 区 两部分｡Main 区就是芯片的程序存储器(ROM),大小为8K ~ 48K;NVR 区的大小为1K｡
Main 区和NVR 区又分为若干Sector 区,每个Sector 为512Byte,也即,Main 区包含16~96 个Sector(48K),NVR 区包含2 个Sector(1K)｡

Flash 存储器在正常工作状态下(整个VDD 范围内)是可 读写的｡这些存储器并不直接映射到寄存器文件空间,而是 通过特殊功能寄存器对其进行间接寻址｡共有5 个特殊功能 寄存器用于访问这些存储器:FDR FADRL FADRH FCR FPSR

## 19.1. sector 擦除
要写Flash 存储器之前,必须要先进行擦除操作｡EFC 模块只支持sector 擦除,也就是每次擦除都会将1 个sector 整个的擦除｡
对于sector 擦除操作,FADRL 和FADRH 组成一个 16 位Flash 存储器地址｡FCR 寄存器的NVRSEL 位控制擦除 Main 区还是NVR 区,ERASE 位用于控制启动sector 擦除｡ 只要设置Flash 存储器地址落在目标sector 内,启动sector 擦除后,就会擦除目标sector｡将ERASE 位置1,即启动sector 擦除,擦除完成后会自动清0,所以需要查询ERASE 位是否 为0,来判断擦除是否完成｡
应注意的是,在写FCR 寄存器时,必须先按顺序写FPSR 等于0x55､0xAA。
```c
//例19-1 sector擦除
void eraseSector(unsigned char sel,unsigned short adr)
{
    FPSR = 0x55;
    FPSR = 0xAA;
    FADRL = adr & 0x00ff;
    FADRH = (adr >> 8);
    FCR = 0x02 | (sel << 2);
    while(FCR & 0x02);
}
```

## 19.2. 字节编程
EFC 功能支持对Flash 存储器字节编程,用户应该先将 编程字节的地址写入FADRL 和FADRH 寄存器,编程数据写入 FDR 寄存器,FCR 寄存器的NVRSEL 位控制擦除Main 区还是 NVR 区,PROG 位用于控制启动字节编程｡将PROG 位置1,即 启动字节编程,编程完成后会自动清0,所以需要查询PROG 位是否为0,来判断编程是否完成｡
```c
//例19-2 字节写入
void writeByte(unsigned char sel,unsigned short adr,unsigned char dat)
{
    FPSR = 0x55;
    FPSR = 0xAA;
    FDR = dat;
    FADRL = adr & 0xFF;
    FADRH = adr >> 8;
    FCR = 0x01 | (sel << 2);
    while(FCR & 0x01);
}
```

## 20.1. LED 的操作步骤
1. 端口配置:

#### 数码类型

MODE：000=4×13、001=5×12、010=6×11、011=7×10、100=9×8、110=10×7；

#### 刷新分频
\[T=Tosc800k \times( DIV1+1) ×2^{DIV2} ×8 ×N ×Y;\quad F=\frac1T\]
DIV1来自LEDDIVR，DIV2来自CKDIVCR.SLOWDIV；

4. 显示数据配置
- 4/5/6/7位规格：LEDDR0/1组成16bit取低位做段；
- 9/10位规格：单DR对应一位段；
**修改数据前：LEDCR.ONOFF=0，等待WREN=1再改写。**

5. 使能显示：ONOFF=1。

# 附录Ⅲ LED扫描示例程序
```c
#include <CBM7316.H>
#include "Display.h"
unsigned char g_Led[4];
code unsigned char Code_SMG_Tab[16]={0x3F,0x06,0x5B,0x4F,0x66,0x6D,0x7D,0x07,0x7F,0x6F,0x77,0x7C,0x39,0x5E,0x79,0x71};

void SegLight(unsigned char num)
{
    unsigned char i;
    for(i=0x01;i<=0x80;i<<=1)
    {
        switch(i&num)
        {
        case 0x01: SEG1 = 1; break;
        case 0x02: SEG2 = 1; break;
        case 0x04: SEG3 = 1; break;
        case 0x08: SEG4 = 1; break;
        case 0x10: SEG5 = 1; break;
        case 0x20: SEG6 = 1; break;
        case 0x40: SEG7 = 1; break;
        case 0x80: SEG8 = 1; break;
        case 0:
            switch(i)
            {
            case 0x01: SEG1 = 0;break;
            case 0x02: SEG2 = 0;break;
            case 0x04: SEG3 = 0;break;
            case 0x08: SEG4 = 0;break;
            case 0x10: SEG5 = 0;break;
            case 0x20: SEG6 = 0;break;
            case 0x40: SEG7 = 0;break;
            case 0x80: SEG8 = 0;break;
            default:break;
            }
            break;
        default:break;
        }
    }
}

void Display(void)
{
    static unsigned char com=1;
    switch(com)
    {
    case 1:
        COM1=1;COM2=1;COM3=1;COM4=1;
        SegLight(Code_SMG_Tab[g_Led[0]]);
        COM1=0;
        com=2;
        break;
    case 2:
        COM1=1;COM2=1;COM3=1;COM4=1;
        SegLight(Code_SMG_Tab[g_Led[1]]);
        COM2=0;
        com=3;
        break;
    case 3:
        COM1=1;COM2=1;COM3=1;COM4=1;
        SegLight(Code_SMG_Tab[g_Led[2]]);
        COM3=0;
        com=4;
```

# 附录Ⅳ 内部48M RC时钟校准代码
```c
volatile unsigned int xdata checkCount = 0,saveCheckCount = 0;
volatile unsigned char xdata CheckDelayCount = 0;
extern unsigned int xdata KeyValue;

void oscCheckCountAdd(void)
{
    checkCount++;
}

void osc48MCheck(unsigned int tCount,unsigned char dCount)
{
    unsigned char n = tCount/200;
    saveCheckCount = checkCount;
    checkCount = 0;
    if(KeyValue==0)
        CheckDelayCount++;
    else
        CheckDelayCount=0;
    if(CheckDelayCount >= dCount)
    {
        if((saveCheckCount + n) < tCount)
        {
            if(0x7F > OSC48MTRIM)
                OSC48MTR +=1;
        }
        else if((saveCheckCount-n)>tCount)
        {
            if(0x00 < OSC48MTRIM)
                OSC48MTR -=1;
        }
        CheckDelayCount=0;
    }
}
```
说明：
1、内部RC时钟受温漂影响，UART应用必须添加实时校准，校准后全温48M误差±2%；
2、使用两个定时器：一个16M计数、一个800K计数；
3、tCount=800K定时/16M定时比值（推荐＞500:1），dCount为校准周期系数，校准间隔5~30S。

## 操作方法
### 一、P0PD寄存器操作
1、P0PD7清零：
字节：`P0PD &= 0x7F;`
位：`P0PD7 = 0x00;`
2、P0PD7置1：
字节：`P0PD |= 0x80;`
位：`P0PD7 = 0x80;`

### 二、P0PURSELRL寄存器（上拉档位）
1、bit3清零：`P0PURSEL &=0x3F; P0PURSELRL3=0;`
2、bit3=1：`P0PURSEL |=0x40; P0PURSELRL3=0x40;`
3、bit3=2：`P0PURSEL |=0x80; P0PURSELRL3=0x80;`
4、bit3=3：`P0PURSEL |=0xC0; P0PURSELRL3=0xC0;`
